## 2. Prompt Engineering 策略设计

### 2.1 核心策略：多轮迭代修正

基于以下考虑设计策略：
1. **max_turns=3 的限制**: 需要高效利用每一轮
2. **测试反馈的价值**: execute_code 提供的 feedback 是改进的关键
3. **错误类型多样**: Runtime Error, Time Limit Exceeded, Wrong Answer 需要不同的处理

### 2.2 策略架构

```
第1轮: 生成初始代码
  ├─ 系统提示词：强调代码质量、边界条件、完整性
  ├─ 用户提示词：原始题目
  └─ 执行测试 → 如果通过，直接返回

第2轮: 根据反馈修正（如果第1轮失败）
  ├─ 分析错误类型（Runtime Error / TLE / Wrong Answer）
  ├─ 提供针对性的修正指导
  └─ 执行测试 → 如果通过，返回

第3轮: 最后一次修正机会（如果第2轮失败）
  ├─ 继续根据反馈修正
  ├─ Token 管理：如果对话过长，保留关键信息
  └─ 返回最终结果
```

### 2.3 系统提示词设计

```
You are an expert competitive programmer. Follow these guidelines:

1. Read the problem carefully and identify:
   - Input/output format
   - Constraints and edge cases
   - Time/space complexity requirements

2. Think step-by-step:
   - Analyze the problem and devise an algorithm
   - Consider edge cases (empty input, large numbers, boundary conditions)
   - Ensure your solution handles all constraints

3. Write clean, efficient Python code:
   - Use appropriate data structures
   - Optimize for time complexity when needed
   - Handle input/output correctly
   - Test with sample cases mentally

4. IMPORTANT: Your code must be complete and runnable.
   Always provide the FULL solution in a ```python code block.
```

**设计理念**:
- 强调"expert competitive programmer"身份，激发模型的专业能力
- 分步骤引导思考过程（问题分析 → 算法设计 → 代码实现）
- 明确要求考虑边界条件和复杂度
- 强调代码完整性（避免只给出部分代码）

### 2.4 错误反馈处理策略

#### Runtime Error 处理
```
The code has a runtime error. Common causes:
- Index out of bounds
- Division by zero
- Type mismatch
- Incorrect variable names
- Missing edge case handling

Please fix the error and provide the corrected COMPLETE code.
```

#### Time Limit Exceeded 处理
```
The code is too slow. Consider:
- Using more efficient algorithms or data structures
- Reducing time complexity (e.g., O(n²) → O(n log n) or O(n))
- Avoiding unnecessary computations
- Using built-in functions when appropriate

Please optimize and provide the COMPLETE code.
```

#### Wrong Answer 处理
```
The code produces incorrect output. Check:
- Logic errors in your algorithm
- Off-by-one errors
- Edge cases (empty input, single element, maximum values)
- Input/output format
- Integer overflow or precision issues

Please fix the logic and provide the corrected COMPLETE code.
```

### 2.5 Token 管理策略

由于单次 prompt 限制为 16k tokens，需要管理对话历史：

```python
total_tokens = sum(count_tokens(msg["content"]) for msg in messages)
if total_tokens > 15000:  # 留一些余量
    # 只保留关键信息
    messages = [
        messages[0],  # system prompt
        messages[1],  # original question
        messages[-2], # last assistant response
        messages[-1]  # current user feedback
    ]
```

**策略**:
- 监控累积 token 数
- 超过阈值时，保留系统提示、原始题目、最近一轮对话
- 丢弃中间的失败尝试，避免上下文污染

## 3. 实现代码

### 3.1 完整实现

```python
def run_question(
    question_prompt: str,
    call_llm,
    execute_code,
    max_turns: int,
    count_tokens,
) -> str:
    # 系统提示词
    system_prompt = """You are an expert competitive programmer. Follow these guidelines:

1. Read the problem carefully and identify:
   - Input/output format
   - Constraints and edge cases
   - Time/space complexity requirements

2. Think step-by-step:
   - Analyze the problem and devise an algorithm
   - Consider edge cases (empty input, large numbers, boundary conditions)
   - Ensure your solution handles all constraints

3. Write clean, efficient Python code:
   - Use appropriate data structures
   - Optimize for time complexity when needed
   - Handle input/output correctly
   - Test with sample cases mentally

4. IMPORTANT: Your code must be complete and runnable. Always provide the FULL solution in a ```python code block."""

    # 第一轮：生成初始代码
    messages = [
        {"role": "system", "content": system_prompt},
        {"role": "user", "content": question_prompt}
    ]

    response = call_llm(messages)
    result = execute_code(response)

    if result["passed"]:
        return response

    # 迭代修正
    best_response = response

    for turn in range(2, max_turns + 1):
        feedback = result["feedback"]

        # 根据错误类型定制反馈
        if "Runtime Error" in feedback:
            error_guidance = """
The code has a runtime error. Common causes:
- Index out of bounds
- Division by zero
- Type mismatch
- Incorrect variable names
- Missing edge case handling

Please fix the error and provide the corrected COMPLETE code."""
        elif "Time Limit Exceeded" in feedback:
            error_guidance = """
The code is too slow. Consider:
- Using more efficient algorithms or data structures
- Reducing time complexity (e.g., O(n²) → O(n log n) or O(n))
- Avoiding unnecessary computations
- Using built-in functions when appropriate

Please optimize and provide the COMPLETE code."""
        elif "Wrong Answer" in feedback:
            error_guidance = """
The code produces incorrect output. Check:
- Logic errors in your algorithm
- Off-by-one errors
- Edge cases (empty input, single element, maximum values)
- Input/output format
- Integer overflow or precision issues

Please fix the logic and provide the corrected COMPLETE code."""
        else:
            error_guidance = """
The code has issues. Please carefully review the error message, fix all problems, and provide the corrected COMPLETE code."""

        messages.append({"role": "assistant", "content": response})
        messages.append({
            "role": "user",
            "content": f"""Your previous solution failed with the following feedback:

{feedback}

{error_guidance}

Remember: Provide the FULL corrected code in a ```python code block, not just the changes."""
        })

        # Token 管理
        total_tokens = sum(count_tokens(msg["content"]) for msg in messages)
        if total_tokens > 15000:
            messages = [
                messages[0],  # system prompt
                messages[1],  # original question
                messages[-2], # last assistant response
                messages[-1]  # current user feedback
            ]

        response = call_llm(messages)
        result = execute_code(response)

        if result["passed"]:
            return response

        best_response = response

    return best_response
```

## 4. 策略优势分析

### 4.1 相比 Baseline 的改进

**Baseline (单轮调用)**:
- 只有一次机会生成代码
- 无法利用测试反馈
- 预期 pass@1 较低

**多轮迭代策略**:
- 最多 3 次机会改进代码
- 利用真实测试反馈定位问题
- 针对不同错误类型提供专门指导
- 预期 pass@1 显著提升

### 4.2 创新点

1. **错误类型分类处理**: 不是简单地把 feedback 传回，而是分析错误类型并提供针对性指导
2. **强调代码完整性**: 多次强调"COMPLETE code"，避免模型只返回修改部分
3. **Token 预算管理**: 主动监控和裁剪对话历史，确保不超限
4. **系统提示词设计**: 激发模型的"竞赛编程专家"角色，提高代码质量

### 4.3 预期效果

- **第1轮成功率**: 预计 40-60%（取决于题目难度）
- **第2轮成功率**: 预计额外 20-30%（通过修正明显错误）
- **第3轮成功率**: 预计额外 5-10%（处理复杂问题）
- **总体 pass@1**: 预计 65-80%（相比 baseline 的 40-50% 有显著提升）

## 5. 潜在改进方向

### 5.1 短期优化

1. **Few-shot 示例**: 在系统提示中加入 1-2 个解题示例（需要权衡 token 开销）
2. **思维链增强**: 要求模型先输出思考过程，再给出代码
3. **测试用例提取**: 从题目中提取样例，要求模型先验证样例

### 5.2 长期探索

1. **自适应策略**: 根据题目类型（图论、DP、贪心等）调整提示词
2. **代码审查机制**: 在提交前让模型自我审查代码
3. **多样性采样**: 生成多个候选解，选择最优的
